# Product Documentation

## Vision
Телеграм-бот для удобного и быстрого трекинга финансов. Основная идея — максимально упростить процесс внесения расходов: пользователь просто пишет или говорит боту, на что потратил деньги, а бот сам структурирует эту информацию.

## Core Features (MVP)

### 1. Ввод данных
- **Текстовые сообщения:** Пользователь пишет в свободном стиле (напр., "оплатила остеопата 10 тысяч рублей").
- **Голосовые сообщения:** Пользователь диктует трату, бот транскрибирует и обрабатывает текст.

### 2. Обработка данных (LLM)
- Распознавание естественного языка (NLP) для извлечения ключевых данных.
- **Целевые поля для базы данных:**
  - **Дата:** Дата совершения траты (по умолчанию — текущая, или указанная в сообщении).
  - **Сумма:** Числовое значение расхода.
  - **Категория:** Автоматическое определение категории (напр., "Здоровье", "Еда", "Транспорт") на основе описания.

### 3. Аналитика и Отчеты
- **Еженедельное Summary:** Сводка трат за неделю.
- **Выбор диапазона:** Возможность запросить отчет за определенные даты.
- **Разбивка по категориям:** Группировка расходов по категориям для анализа структуры трат.

## User Flow
1. Пользователь отправляет сообщение (текст/голос). В приоритете голос для упрощения жизни пользователя.
2. Бот обрабатывает сообщение через LLM.
3. Бот сохраняет запись в БД (Дата, Сумма, Категория). (в дальнейшем возможно интеграция с google таблицами)
4. Бот отправляет подтверждение ("Записано: Остеопат, 10 000 руб., Категория: Здоровье").
5. Пользователь может запросить отчет командой или кнопкой.
